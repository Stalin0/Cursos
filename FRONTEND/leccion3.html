JavaScript y el modelo declarativo

El concepto que desbloquea todo framework: cambias el estado, la vista se redibuja sola.

Cambia el estado, no la pantalla

Estos controles solo modifican un objeto estado y llaman a render(). Nadie toca el DOM a mano. La tarjeta de la derecha es siempre una función del estado.

ver la función render() 
function render(estado) {
  // devuelve la vista a partir del estado. punto.
  mount.innerHTML = `
    <div class="rname">${estado.nombre}</div>
    <div class="rrow">score: ${estado.score}</div>
    <span class="rbadge ${estado.status}">...</span>
    ${estado.productos.map(p => `<span>${p}</span>`).join('')}
  `;
}
// cada control hace: estado.x = ...; render(estado);

El ciclo de vida de una Promise

Promise ≈ Task de C#. Una promesa nace pendiente y termina resuelta o rechazada. Dispara una "petición" simulada y míralo.

pending → fulfilled / rejected
esperando…
async function cargarCliente(id) {
  try {
    const res = await fetch(`/api/clientes/${id}`);  // igual que await en C#
    const data = await res.json();
    return data;
  } catch (e) {
    console.error('falló la petición', e);
  }
}
